На Момент Написания Этой Статьи Заплатка для Работы HELPа была недоступна для Windows 7 и Windows Server 2008 на Сайте MicroSoft с обещаниями выложить при выходе Релиза:
HTTP://Support.MicroSoft.Com/kb/917607
Для Работы Windows Help в Windows 7 необходимо переписать Файлы WinHlp32.Exe и WinHlp32.Exe.MUI в Директорию Windows.
Взять Их можно отсюда:
HTTP://WWW.CrazyMihey.Pisem.Net/WinHlp32_Windows7.Rar
Файлы взяты из Русской Версии Windows Vista, но они будут замечательно работать под Windows 7.
 1. Открываем Директорию %SystemRoot% - в Адресной Строке Проводника (Win+E) вводим "%SystemRoot%"
 2. Находим там Файл "WinHlp32.Exe", нажимаем на Него правой кнопкой Мыши, выбираем Properties.
 3. Переходим на Вкладку Security.
 4. Нажимаем Кнопку Advanced и переходим на Вкладку Owner.
 5. Нажимаем Кнопку Edit...
 6. По Умолчанию Владельцем Файла является TrustedInstaller, что не позволяет Нам переписать (переименовать) Этот Файл, для этого Нам необходимо выполнить следующий Шаг:
 7. В Списке выбираем Группу Administrators и нажимаем Кнопку OK, подтверждаем Наши Действия.
 8. Далее на Вкладке Owner нажимаем Кнопку OK и возвращаемся на Вкладку Security, где Мы теперь можем добавить Права для Группы Administrators.
 9. Нажимаем Кнопку Edit...
10. В Списке выбираем Группу Administrators и ставим Галку напротив Правила Modify (и, автоматически, Write) в Колонке Allow.
11. Нажимаем Кнопку OK.
12. Нажимаем Кнопку OK на Вкладке Security.
13. Спокойно переименовываем Файл "WinHlp32.Exe" во что-нибудь типа "WinHlp32.Exe_".
14. Копируем сюда Файл "WinHlp32.Exe" из этого Архива.
15. Теперь остаётся проделать то же самое с Файлом "WinHlp32.Exe.MUI" в ПодДиректории "en-US" и "ru-RU", если таковая у Вас имеется.
16. После выполнения этих Действий, щёлкаем по любому Файлу с Расширением ".Hlp" и непременно Радуемся Старому Доброму HELPу, который используется огромным количеством Программ.
